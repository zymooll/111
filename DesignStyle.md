1. 整体视觉语言
风格关键词： 极简主义、功能主义、现代、充满活力、高效率、干净。

设计理念： 设计应隐形，让食物和评价本身成为主角。所有的交互和布局都应优先考虑功能的直观性和高转化率。

适配 Ant Design Mobile (ADM)： 设计风格应基于 ADM 的默认组件，并通过主题定制（Custom Theme）来注入“Campus Foodie”的活力，而不是从零开始创建。

2. 颜色方案 (Theme Color)
我们将定义两套模式，以满足不同场景和草图暗示的“深色背景”。

浅色模式（默认，充满活力）
背景： White (#FFFFFF) 或非常淡的灰 (#F8F9FA)，保持通透感。

主色 (Primary)： 一种富有活力但不刺眼的颜色。

推荐 1（学术蓝）： (#007BFF)，代表信任和效率。

推荐 2（活力绿）： (#28A745)，代表健康和餐饮。

强调色 (Accent)： 用于收藏星标、评分。

收藏/评分星标 (★)： 经典的 Amber Yellow (#FFBF00)。

文本：

标题和正文： (#212529) (柔和的黑)。

次要信息： (#6C757D) (中灰)。

深色模式（深夜食堂，草图暗示）
背景： Rich Black (#121212) 或 Dark Gray (#1E1E1E)，避免纯黑以减少眼部疲劳。

主色 (Primary)： 主色的较浅、较低饱和度变体（e.g., 轻学术蓝 #4DA3FF)。

文本： (#E1E1E1) (淡灰白)。

3. 字体体系 (Typography)
系统默认： 使用系统默认字体，确保最佳性能。

推荐： 苹方 (PingFang SC) / 微软雅黑 (Microsoft YaHei)。

字号层级：

一级标题： 18px - 20px (e.g., 商家名称)。

二级标题/加粗： 16px (e.g., 菜品名称)。

正文： 14px (e.g., 评价内容、个性化理由)。

次要/注释： 12px (e.g., 评分数、营业时长)。

字重： 巧妙使用加粗来区分信息层次。

4. 关键页面元素设计（基于 `docs/wireframe.png` 的优化）
你的草图为我们奠定了良好的布局基础。我们将对其进行细化和现代化。

顶部导航 (Top Nav)
优化： 将草图中的垂直排列改为水平排列，以节省屏幕空间。

用户头像： 位于左侧，圆形，带有一个精致的边界。点击进入个人中心。

搜索栏： 位于右侧，现代、圆角矩形，集成搜索图标。占位符文本：“你想吃什么？”

列表卡片 (List Feed Cards)
结构： 遵循草图的左右结构。

左侧 (媒体)： 菜品大图（高质量、圆角），下方是收藏星标 (★)。

右侧 (信息)：

顶部： 菜品名称（加粗）。

下方： 商家名称、评分。

最下方： 个性化推荐理由（使用 Pydantic 过滤出的 JSON 数据，例如“根据您的口味：酸辣”）。

交互： 点击卡片进入详情页；点击星标收藏。卡片应有 subtle 的 Hover 和 Active 状态。

地图页 (Map Page)
高德地图自定义样式： 使用自定义的高德地图样式（例如“清新”或“极夜”），并增加我们的品牌主色和星标 (★)。

标记：

普通商家： 现代化的位置标记。

收藏商家： 标记上有一个明显的星标 (★)。

聚合点： 圆形，包含数字（商家数量），如果包含收藏商家，则聚合圈外围也显示星标。

底部导航 (Bottom Nav)
布局： 根据最新产品确认，首页底栏采用三等分布局，只显示“首页、地图、我的”。

组件： 使用简洁的底部 TabBar，并适配移动端安全区。

评价入口： “我也吃过”不出现在首页底栏，只保留在菜品详情页底部，避免与一级导航混淆。

5. 交互与动效
极简动效： 动效仅用于引导和提供反馈。

转换： 滑动加载 (TanStack Query 优化)、卡片展开、地图缩放。

加载： 使用 ADM 的 Skeleton 组件（骨架屏）来优化首屏加载体验。

6. PWA 与离线
可安装： 界面应感觉像原生应用。
